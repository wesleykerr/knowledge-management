## Notes

### Knowledge-listener scope reduction

arxiv and youtube processors were removed from the listener pipeline entirely:

- **Why:** Both processors already fetched their own content independently (arxiv via the arxiv API + PDF download, youtube via YouTube Data API). The HTML payload written to `~/sync/inbound/` was never used by either processor — passing it was pure waste.
- **Decision:** Filter these URL types at the API boundary. `api_server.py` now checks `"arxiv.org" in url.lower() or "youtube.com" in url.lower()` before writing any file. Returns `{"success": true, "skipped": true}` immediately.
- **Listener** now only handles two cases: twitter → `twitter.process_twitter_url()`, everything else → `default.process_url()`.
- `doc_migration.py` was a one-off migration script that imported the now-deleted arxiv processor — caught by the code quality reviewer and deleted.
- Removing `marker-pdf` dropped 78 packages from the lockfile (torch, transformers, surya-ocr, etc.). `markdownify` and `anthropic` were promoted to explicit dependencies since they had only been transitive under marker-pdf.

## Module restructure and Mac deployment

Third session of the day completed the module restructure and deployment work.

### Package rename

`bookmarks/` → `knowledge/`, `processors/default.py` → `processors/webpage.py`. All internal imports updated throughout. `api_server.py` moved into the package as `knowledge/server.py` with a `main()` entry point added.

### Entry points

Added `[project.scripts]` to `pyproject.toml`:
- `knowledge-server = "knowledge.server:main"`
- `knowledge-listener = "knowledge.listener:main"`

Required adding a `[build-system]` block and `[tool.setuptools.packages.find]` with `include = ["knowledge*"]` — without a build system uv doesn't install the package in editable mode and the entry point scripts are never created.

### Deployment

Split deployment clarified during implementation:
- **Linux** — `knowledge.service` (systemd): runs the gunicorn API server. Updated from miniconda + `api_server:app` to `uv run gunicorn knowledge.server:app`.
- **Mac mini** — `com.wkerr.knowledge-listener.plist` (launchd): runs the listener. Sources `.env` via `run-with-env.sh` wrapper before handing off to uv.
- `Makefile` on the Mac mini manages the listener plist (`make install` / `make start` / `make stop` / `make logs`).

### Dependency cleanup

Removed 9 unused packages (absl-py, fastapi, uvicorn, langchain, pymupdf4llm, replicate, selenium, sqlalchemy, tweepy). Promoted 4 to explicit direct deps (gunicorn, requests, jinja2, httpx). Net: ~32 fewer packages in the lockfile.
