## Notes

### Deployment script

Created `scripts/cli.sh` to automate deployment to the Linux server (172.245.131.124):

1. Tars `knowledge/`, `scripts/`, `pyproject.toml`, `.python-version` into `/tmp/code.tgz`
2. SCPs the tarball to the server
3. Extracts into a timestamped directory (`knowledge-management-<uuid-suffix>`)
4. Symlinks `~/knowledge-management` to the new directory
5. Copies the systemd service file and restarts the service

**SSH + sudo fix:** The initial version piped commands via `ssh ... < /tmp/code.txt`, which prevented TTY allocation for sudo password prompts. Fixed by passing the script as an ssh argument (`ssh -t ... "$REMOTE_SCRIPT"`) so `-t` can allocate a pseudo-TTY.
